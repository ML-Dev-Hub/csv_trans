  <section class="home-props not-content" aria-label="Why csv-trans">
    <div class="home-prop">
      <h2 class="home-prop__title">Byte-preserving by design</h2>
      <p class="home-prop__body">
        Delimiter, quoting, encoding, line endings, row order, and field counts
        are detected and reproduced exactly. Every run ends in an atomic write.
      </p>
</div>
    <div class="home-prop">
      <h2 class="home-prop__title">Zero dependencies</h2>
      <p class="home-prop__body">
        Pure standard library on Python 3.11+. Nothing to pin, audit, or
        resolve — <code>pip install csv-trans</code> is the whole supply chain.
      </p>
</div>
    <div class="home-prop">
      <h2 class="home-prop__title">Enforced privacy modes</h2>
      <p class="home-prop__body">
        <code>public</code>, <code>restricted</code>, and <code>local-only</code>
        boundaries are validated before any text leaves the machine and
        re-checked on every provider call, including retries and fallbacks.
      </p>
</div>
    <div class="home-prop">
      <h2 class="home-prop__title">Bounded recovery</h2>
      <p class="home-prop__body">
        Failed batches walk a fixed ladder — retry, corrective retry, split,
        adaptive singletons, provider fallback. A cell that still fails keeps
        its original text and is reported, never dropped.
      </p>
</div>
  </section>
